5. <meta name="viewport" content="width=device-width, initial-scale=1.0">
_______________________________________________________________________
a. width=device-width — Sets the viewport width to match the device's screen width.
b. initial-scale=1.0 — Sets the initial zoom level to 100% when the page first loads.
c. viewport — The visible area of a web page in the browser (display area). The viewport meta tag is used to configure how this area is displayed on different devices.

10. HTML List Elements
___________________________
HTML provides three main types of lists:

1. Ordered List (<ol>)

Used when the order of items matters.

<ol>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ol>

1. HTML

Used when the order of items does not matter.

<ul>

Apple

13. HTML Image Element (<img>) with Folder Concepts

project/
 ├── index.html

project/
 ├── index.html

project/
 ├── image.jpg

<img src="https://example.com/photo.jpg" alt="Online Image">

Stylesheet Languages
_____________________

A stylesheet language is used to control the presentation and appearance of HTML

Common Stylesheet Languages:

1. CSS (Cascading Style Sheets)
The standard stylesheet language for web pages.
Used to style HTML elements (colors, fonts, layouts, spacing, etc.).

Example:
h1 {
  color: blue;
}

2. Sass (Syntactically Awesome Style Sheets)
A CSS preprocessor that extends CSS with features like variables, nesting, and mixins.
Compiles into regular CSS.

Example:
$color: blue;

h1 {
  color: $color;
}

3. Less (Leaner Style Sheets)
Another CSS preprocessor similar to Sass.
Adds variables, functions, and nesting.
Compiles into regular CSS.

Example:
@color: blue;

h1 {
  color: @color;
}

How To Css in HTML
____________________

We can do that in 3 ways

1. Inline CSS

-> CSS is written directly inside an HTML element using the style attribute. Style attribute is globally available for HTML elements tags like img, html, h1 etc.

-> Syntax to add style attribute: name="property:value"

-> Should be applied individually to each elements and not recommended for production use, should be used only for quick testing purpose

Example 1:

Example 2:

</html>

CSS is written inside a <style> tag in the <head> section. Use For styling a single web page

Syntax:

<head>
    <style>
        selector {
            property: value;
            property: value;
        }
    </style>
</head>



<!DOCTYPE html>

Font Size:

How to set font size?

1 px = 1/96th inch = 0.26mm (width and height)

So for, 20px = 20*0.26 = 5.2 mm (width and height)


also we can set the font as 1 pt(point)

1 pt-->1/72 inches = 0.35 mm
so, 20 pt = 0.35*20 = 7.0 mm

em is a relative unit. It means:

1em = the current font size of the element's parent (for font-size).

1em = parent's font size
     = 20px

2 em = parents font size*2

Example:

style.css

html{
    background-color: red;
}
div{
    font-size: 20px;
}

p{
    font-size: 2em;
}

index.html
----------
<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <link rel="stylesheet" href="style.css" />
</head>

<body>
    <div class="parent">
        <p class="child">Hello</p>
    </div>

</body>

</html>


What is 1rem in CSS?

rem stands for Root EM.

1rem = font size of the root element (<html>)

Default case

Most browsers set:

html {
    font-size: 16px;
}

So:

1rem = 16px

Example 1

html {
    font-size: 16px;
}

p {
    font-size: 1rem;
}

Result:

1rem = 16px

So paragraph text = 16px.

Example 2

html {
    font-size: 16px;
}

h1 {
    font-size: 2rem;
}

Calculation:

2rem = 2 × 16px = 32px

################################
Note:
If everything was fixed in px, then:

A phone screen
A laptop
A 4K monitor

would all show text the same size.

That causes problems:

Too small on big screens
Too large on small screens
so, we use em and rem
##################################################

❌ Problem: Using only px (fixed design)
HTML
<div class="container">
    <h1>Welcome</h1>
    <p>This is a paragraph.</p>
    <button>Click Me</button>
</div>

CSS (bad approach)
html {
    font-size: 16px;
}

h1 {
    font-size: 32px;
}

p {
    font-size: 16px;
}

button {
    font-size: 16px;
    padding: 10px 20px;
}
🚨 What goes wrong?

Now imagine user changes browser setting:

“Increase default text size for better readability”

OR uses zoom / accessibility settings.

Result:
Text becomes harder to scale consistently
Buttons feel too small
Spacing feels off
You must manually adjust every value

✅ Solution: Using rem (scalable design)
CSS (good approach)

html {
    font-size: 16px;
}

h1 {
    font-size: 2rem;   /* 32px */
}

p {
    font-size: 1rem;   /* 16px */
}

button {
    font-size: 1rem;
    padding: 0.625rem 1.25rem;
}
🔥 Now the magic part

If we change just ONE line:

html {
    font-size: 20px;
}
Everything scales automatically:
Element	Before	After
1rem text	16px	20px
h1 (2rem)	32px	40px
button text	16px	20px
padding	scaled	scaled
📱 Why this is powerful

Now your UI:

adjusts for mobile users
supports accessibility settings
scales consistently everywhere
requires no redesign

More About Device pixels
#############################

-> Device pixel = Total number of pixel in the screen
Example: 6 inches screen has 1800 pixels

-> Pixel per Inch = How dense the pixel are in 1 inch

example:
ppi = total screen pixels / screen size

Example: ppi = 1800/6 = 300 pixels

More pixels per inch means more sharp images

Note: CSS pixel is font size in browser.

Depending on the screen size font size automatically gets adjusted.

The viewport is the visible area of a web page on your screen.

In simple terms:
👉 It is the part of the page you can see without scrolling.

🖥️ Example

When you open a website:

Your screen shows only a portion of the page
That visible portion = viewport

If you scroll down, the viewport changes.

👉 The viewport is the visible area of the web page inside the browser window.

And yes:

vw = viewport width (1% of visible width)
vh = viewport height (1% of visible height)

So when the browser size changes:
✔ vw and vh automatically adjust
✔ Layout becomes responsive

Note:

font-size:
a. Applies only to text
b. Controls how big letters appear

Viewport (screen-related)
Viewport = visible browser area

Then we use:

vw → % of viewport width
vh → % of viewport height

Example:

style.css
----
div{
    background-color: blue;
    width: 50vw;
    height: 50vh;
    font-size: 30px;
}

How to develop responsive websites
------------------------------------

We can do this in various ways
1. media query: Media queries let you apply different CSS rules based on the device or viewport size. They are one of the main tools used to make websites responsive.

@media(max-width: 600px){
  /* CSS for scree below or equal to 600 px */
}

Example:
style.css
---------
header {
    background-color: blue;
    color: white;
    height: 10vh;
    text-align: center;
    line-height: 10vh;
    font-size: 24px;
}

@media (max-width: 600px) {
    header {
        font-size: 16px;
    }
}

index.html
---------
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <header>
        My Website
    </header>


</body>
    
</body>
</html>

    gap: 10px;

.box {
    background-color: blue;
    color: white;
    padding: 20px;
    text-align: center;
}

.first{
    background-color: blue;
    color: white;
    padding: 20px;
   grid-column: span 2 ;

3. CSS Flexbox
------------
CSS Flexbox (Flexible Box Layout) is a layout system used to arrange items in a single direction (row or column) and easily control spacing, alignment, and order.

🆚 Flexbox   vs  Grid (simple)
-----------------------------

One direction	Two directions
Row OR column	Row AND column
Simple layouts	Complex layouts

Important note: Both Grid and flex can be used together

CSS Framework developer by twitter developer (Mark otto and Jacob)

Bootstrap is a ready-made collection of CSS and JavaScript components that helps you design websites faster.

🔥 Why Bootstrap is used

Instead of writing everything from scratch like:

Flexbox
Grid
Buttons
Forms
Navbar

👉 Bootstrap already gives you all of these pre-built.

Note:
A CDN is a system of servers that delivers files (like CSS, JS, images) to users from the nearest location for faster loading.

🧠 Why CDN is used
✔ Faster loading

Files come from nearby servers

✔ No download needed

You don’t install Bootstrap manually

✔ Always updated

You get latest version easily

✔ Less load on your server

Your website doesn’t carry heavy files

📦 What does mb-3 mean?
m  = margin
b  = bottom
3  = size (Bootstrap spacing scale)


🧱 1. <div>

A simple container element used to hold content.

Here it is acting like a product card box.

🎯 2. p-3 (Padding)
p = padding
3 = Bootstrap spacing size
Meaning:

Adds padding inside the box on all sides.

Small space inside the box:
| Product 1 |

Bootstrap scale:

p-1 = small padding
p-3 = medium padding
p-5 = large padding
🎨 3. bg-light (Background color)

Gives a light gray background.

Light background box:
┌──────────────┐
│ Product 1    │
└──────────────┘

It is useful to highlight content softly without strong colors.

🧾 4. border

Adds a thin border around the box.

┌──────────────┐
│ Product 1    │
└──────────────┘

Without border → flat look
With border → card-like structure

🧠 Final Combined Effect
┌──────────────────┐
│ Product 1        │
└──────────────────┘
p-3 → gives spacing inside
bg-light → makes background light gray
border → adds outline

Virtual scrolling, better CLI prompts.
Angular 8 (2019)
Differential loading, lazy loading improvements.
Angular 9 (2020)

Angular 10–13 (2020–2021)
Improvements in performance, strict typing, updates to tooling.
Angular 14–15 (2022)

Angular 16–17 (2023–2024)
Signals introduced (reactivity model improvements), better SSR.

Flow Control Statements in TypeScript

Flow control statements determine the order in which code executes.

1. if Statement

Executes a block if a condition is true.

Example:

let age: number = 20;

if (age >= 18) {
    console.log("You are an adult");
}

Output:

You are an adult


2. if...else Statement

Executes one block if the condition is true, otherwise another block.

Example:
---------
let age: number = 16;

if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}

Output:

Minor

3. if...else if...else

Used to check multiple conditions.

Example:

let marks: number = 85;

if (marks >= 90) {
    console.log("Grade A");
} else if (marks >= 75) {
    console.log("Grade B");
} else {
    console.log("Grade C");
}

Output:

Grade B

4. switch Statement

Used when comparing one value against multiple cases.

Example:

let day: number = 3;

switch (day) {
    case 1:
        console.log("Monday");
        break;
    case 2:
        console.log("Tuesday");
        break;
    case 3:
        console.log("Wednesday");
        break;
    default:
        console.log("Invalid day");
}

Output:

Wednesday

5. for Loop

Repeats code a specific number of times.

Example

for (let i = 1; i <= 5; i++) {
    console.log(i);
}

Output:

1
2
3
4
5


6. while Loop

Repeats while a condition is true.

Example:

let i: number = 1;

while (i <= 3) {
    console.log(i);
    i++;
}

Output:

1
2
3

7. do...while Loop

Executes the block at least once.

let i: number = 1;

do {
    console.log(i);
    i++;
} while (i <= 3);

Output:

1
2
3

8. break Statement

Stops a loop or switch.

for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        break;
    }
    console.log(i);
}

Output:

1
2

9. continue Statement

Skips the current iteration and moves to the next one.

for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        continue;
    }
    console.log(i);
}

Output:

1

Signals in Angular
---------------------

Option 1: zone.js is a library (Angular 2) that helps Angular know when something has changed so it can automatically update the UI.

Button Click / HTTP Response / Timer
                ↓
            Zone.js
                ↓
    Angular Change Detection
                ↓
Checks component tree
                ↓
Updates UI where values changed

Option 2: Signals(Angular 16 and above)
--------------------------------------
Think of a signal as a variable that Angular automatically tracks. When the signal value changes, Angular knows exactly what needs to be updated in the UI.

count signal changed
        ↓
Angular knows which component/template uses count
        ↓
Updates only that part of the UI

In the context of UI updates and change detection, Signals are generally more efficient than the traditional Zone.js-based approach.
